LLM 执行模式

面向连通性测试、正式报告生成和可复现评分的 Kimi 分层策略。

模式

模式用途Kimi 配置理由
Smoke test验证 API 连接thinking disabled，temperature=0.6快且成本低。
正式报告生成报告章节thinking enabled，temperature=1.0，max_tokens=8192保留质量，同时控制延迟。
评分Rubric 评测默认不用 LLM可复现、可审计。

观察发现

  • Kimi thinking 成功生成 NVDA 章节，但有延迟长尾。
  • News 数据缺失被披露，而不是编造。
  • 估值段落出现了与源 CSV 不一致的数字。
  • FMP endpoint 和日志脱敏问题成为 harness finding。
